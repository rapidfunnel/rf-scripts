RF Scripts v1 — Developer Documentation

Client-side jQuery scripts for RapidFunnel-powered Webflow pages  ·  Repository: rf-scripts/v1  ·  Last updated: February 2026
Overview: All scripts in this library are designed to be embedded in Webflow pages served by the RapidFunnel platform. They read the URL query parameters userId, resourceId, and contactId to personalize behavior, and communicate with the RapidFunnel REST API to fetch user data, submit contacts, and send notifications. All scripts require jQuery to be loaded on the page.

analyticsDetails.js

Analytics Details

Purpose: Dynamically injects third-party analytics tracking codes (Google Analytics, Facebook Pixel, Google Tag Manager) fetched from the RapidFunnel API based on the current user.

How It Works

  1. Reads userId from the page URL query string.
  2. Calls GET https://apiv2.rapidfunnel.com/v2/analytics/{userId}.
  3. The response contains userData and accountData objects, each optionally holding tracking codes.
  4. Injects each tracking snippet into <head> if not already injected (deduplication via Set).

Supported Tracking Providers

ProviderAPI FieldInjection Method
Google Analytics 4googleTrackingCodeAppends gtag.js + init script to <head>
Facebook PixelfbTrackingCodeAppends standard fbevents.js snippet to <head>
Google Tag ManagergtmTrackingCodeAppends GTM loader script to <head>

Dependencies

jQuery URL param: userId

Required HTML

None — the script runs on DOM ready with no required markup.

bookMe.js

Book Me Button

Purpose: Controls visibility of the "Book Me" button based on whether a valid booking link has been loaded by userDetails.js.

How It Works

  1. Hides #bookMeBtn on page load.
  2. Listens for the custom event customBookingLinkUpdated (fired by userDetails.js once the booking link is populated).
  3. Calls checkAndShowButton() on load and on every event trigger.
  4. Shows the button only if its href is a non-empty, non-# value.

Dependencies

jQuery userDetails.js

Required HTML

<a id="bookMeBtn" href="#">Book Me</a>
branding.js

Branding

Purpose: Fetches a user's branding settings from the API and applies their custom colors to page elements. Also controls the "Powered by RF" logo visibility.

How It Works

  1. Reads userId from the URL.
  2. Calls GET https://app.rapidfunnel.com/api/branding/user/{userId}.
  3. Sets the background CSS property on elements matching the class names below.
  4. Shows #pbrf_logo by default; hides it if hidePoweredByRFLogoOnResources === true.

CSS Class Mappings

API FieldCSS Class to Target
primaryColor.rf_primaryColor
primaryColorOffset.rf_primaryColorOff
secondaryColor.rf_secondaryColor
secondaryColorOffset.rf_secondaryColorOff
tertiaryColor.rf_tertiaryColor
tertiaryColorOffset.rf_tertiaryColorOff

Dependencies

jQuery URL param: userId

Required HTML

Any elements using the CSS classes listed above, and optionally <div id="pbrf_logo">.

contactForm.js

Contact Form

Purpose: Handles a contact capture form — pre-populates fields for known contacts, validates input, and submits data to the RapidFunnel API.

How It Works

  1. Reads userId, resourceId, and contactId from the URL.
  2. If contactId is present, pre-populates the form by calling GET /v2/contact-details/{contactId}.
  3. On submit button click (#contactFormSubmitBtn):
    • Validates that First Name, Last Name, Email, and Phone are all filled.
    • Disables the button and shows "Submitting..." text.
    • Posts to https://my.rapidfunnel.com/landing/resource/create-custom-contact.
    • Campaign and label IDs are read from global variables assignCampaignId and labelId.
  4. On success, redirects using the URL in data-redirect on #contactFormSubmitContainer, replacing [user-id], [resourceID], and [contactId] placeholders.

Dependencies

jQuery URL params: userId, resourceId, contactId Global vars: assignCampaignId, labelId

Required HTML

<div id="contactFormSubmitContainer"
     data-redirect="https://example.com?userId=[user-id]&contactId=[contactId]">
  <input id="contactFirstName"  type="text">
  <input id="contactLastName"   type="text">
  <input id="contactEmail"      type="email">
  <input id="contactPhone"      type="tel">
  <button id="contactFormSubmitBtn">Submit</button>
</div>
contactFormGdpr.js

Contact Form — GDPR

Purpose: A two-part script that handles GDPR-compliant contact form submission. Part 1 validates the form and consent fields; Part 2 creates the RapidFunnel contact via API. Both parts are defined in the same file and communicate through shared global flags on window.

Part 1 — Validation

  1. Listens for the click event on #contactFormSubmitBtn.
  2. Sets window.isValidating = true to prevent duplicate triggers.
  3. Validates that First Name, Last Name, Email, and Phone are all filled.
  4. Validates that the #gdprConsent checkbox is checked.
  5. If validation fails: shows inline error messages and resets window.isValidating = false.
  6. If validation passes: sets window.validationComplete = true and calls window.createContactAfterValidation().

Part 2 — Contact Creation

  1. Checks window.validationComplete on load — exits immediately if false.
  2. Exposes window.createContactAfterValidation() globally, invoked by Part 1.
  3. On invocation:
    • Double-checks window.validationComplete.
    • Reads userId, resourceId from the URL; reads data-campaign and data-label from #contactFormContainer.
    • Disables the submit button and shows the spinner.
    • Posts form data to https://my.rapidfunnel.com/landing/resource/create-custom-contact.
    • On success: redirects with userId, resourceId, and new contactId appended; or clears the form (including #gdprConsent and #marketingConsent) if no redirect URL is set.
  4. Uses isSubmitting flag to prevent duplicate API calls.
  5. Resets window.validationComplete and window.isValidating after each attempt.

Dependencies

jQuery URL params: userId, resourceId

Required HTML

<div id="contactFormContainer"
     data-campaign="CAMPAIGN_ID"
     data-label="LABEL_ID"
     data-redirect="https://example.com/thank-you">
  <input id="contactFirstName">
  <input id="contactLastName">
  <input id="contactEmail">
  <input id="contactPhone">
  <input id="gdprConsent"      type="checkbox">
  <input id="marketingConsent" type="checkbox">
  <button id="contactFormSubmitBtn">
    <span class="btn-text">Submit</span>
    <span class="spinner" style="display:none"></span>
  </button>
</div>
contactFormv2.js

Contact Form v2 (Updated)

Purpose: An updated version of contactFormGdpr.js, also a two-part script. Adds a stricter second validation flag (window.VALIDATION_PASSED) and uses a more generic submit button selector instead of a hard-coded ID.

Part 1 — Validation

  1. Listens for the submit event on the form inside .form-container.
  2. Sets window.isValidating = true to prevent duplicate triggers.
  3. Validates that First Name, Last Name, Email, and Phone are all filled.
  4. Validates that the #gdprConsent checkbox is checked.
  5. If validation fails: shows inline error messages and resets window.isValidating = false.
  6. If validation passes: sets both window.validationComplete = true and window.VALIDATION_PASSED = true, then calls window.createContactAfterValidation().

Part 2 — Contact Creation

Behaves the same as contactFormGdpr.js Part 2 with the following differences:

FeaturecontactFormGdpr.jscontactFormv2.js
Validation gatewindow.validationComplete onlywindow.validationComplete AND window.VALIDATION_PASSED
Submit button selector#contactFormSubmitBtn (by ID).form-container form button[type="submit"] (generic)
GDPR fields cleared on successYes (#gdprConsent, #marketingConsent)No
VALIDATION_PASSED reset after submitNot applicableYes — window.VALIDATION_PASSED = false

Dependencies

jQuery URL params: userId, resourceId

Required HTML

Same container structure as contactFormGdpr.js. The submit button only needs to be a button[type="submit"] or input[type="submit"] inside .form-container form — no specific ID required.

ctaButton.js

CTA Button

Purpose: Intercepts clicks on CTA buttons, sends a notification email to the user with the contact's details and page context, then redirects the visitor to the button's linked URL.

How It Works

  1. Reads userId, resourceId, and contactId from the URL.
  2. Attaches a click handler to all elements whose ID starts with ctaButton.
  3. On click:
    • If contactId is present: fetches contact details from /v2/contact-details/{contactId}, then POSTs to https://app.rapidfunnel.com/api/mail/send-cta-email with name, phone, email, button description (data-description), and page name.
    • Redirects to the button's href (in a new tab if target="_blank") regardless of whether the email succeeded.

Dependencies

jQuery URL params: userId, contactId

Required HTML

<a id="ctaButton1"
   href="https://example.com"
   target="_blank"
   data-description="Main Hero CTA">Get Started</a>
Note: Multiple buttons are supported — use incrementing IDs: ctaButton1, ctaButton2, etc.
ctaConversion.js

CTA Conversion Button

Purpose: Similar to ctaButton.js but uses the conversion-specific email endpoint (send-cta-conversion-email). Always sends an email even when no contact is found, using placeholder values.

How It Works

  1. Reads userId and contactId from the URL.
  2. Listens for clicks on elements with IDs starting with ctaConversionButton.
  3. On click:
    • If contactId present: fetches contact details and sends email with real data.
    • If contact fetch fails: sends email with "System failed to answer" as placeholder name.
    • If no contactId: sends email with "No contact ID found" as placeholder name.
  4. Always redirects to the button's href after the email attempt.

Dependencies

jQuery URL params: userId, contactId

Required HTML

<a id="ctaConversionButton1"
   href="https://example.com/download"
   data-description="Download PDF">Download</a>
ctaResourceButton.js

CTA Resource Button

Purpose: Handles CTA buttons that link to RapidFunnel resources. Dynamically resolves and sets the button href from the API on page load, then sends a tracking email before redirecting on click.

How It Works

  1. Reads userId and contactId from the URL.
  2. On load, iterates all elements with IDs starting with ctaResourceButton:
    • Reads data-cta-resource-id from each button.
    • Calls GET /api/api/resources/resource-details/?userId=...&resourceId=...&contactId=....
    • Sets the resolved URL (+ userId and optional contactId as path segments) as the button's href.
    • Disables the button on API failure.
  3. On click: fetches contact details and POSTs to send-cta-email, then redirects.

Dependencies

jQuery URL params: userId, contactId

Required HTML

<a id="ctaResourceButton1"
   data-cta-resource-id="123"
   data-cta-res-location="Hero Section"
   href="#">View Resource</a>
placeholderScript.js

Placeholder Script

Purpose: A general-purpose utility that resolves URL token placeholders in every <a> tag's href on the page. Useful for pages built with static links that need dynamic IDs injected at runtime.

How It Works

  1. Reads userId, resourceId, and contactId from the URL.
  2. Iterates every <a> element on the page.
  3. Replaces the following tokens in each href value:
TokenReplaced WithFallback
[user-id]Value of userId URL paramEmpty string
[userId]Value of userId URL paramEmpty string
[contactId]Value of contactId URL paramEmpty string

Dependencies

jQuery URL params: userId, contactId

Usage Example

<!-- Before script runs -->
<a href="https://example.com?u=[user-id]&c=[contactId]">Link</a>

<!-- After script runs (e.g., userId=42, contactId=99) -->
<a href="https://example.com?u=42&c=99">Link</a>
requestCallForm.js

Request Call Form

Purpose: Handles a "Request a Call" interaction. Adapts its display based on whether a contact is already known, and sends an email notification to the assigned user.

How It Works

  1. Reads userId, resourceId, and contactId from the URL.
  2. Display logic on load:
    • If contactId present → show #requestCallBtn, hide #requestCallForm.
    • If no contactId → show #requestCallForm, hide #requestCallBtn.
  3. Button mode (known contact): On click, fetches contact details then POSTs to send-request-call-email.
  4. Form mode (anonymous): On form submit, reads First Name, Last Name, and Phone from the form inputs and POSTs to send-request-call-email. Shows .w-form-done on success, .w-form-fail on error.

Dependencies

jQuery URL params: userId, contactId Global var: resourceDescriptionForRequestCall

Required HTML

<button id="requestCallBtn">Request a Call</button>

<div id="requestForm">
  <form id="requestCallForm">
    <input id="contactFirstNameRequestForm">
    <input id="contactLastNameRequestForm">
    <input id="contactPhoneRequestForm">
  </form>
  <div class="w-form-done" style="display:none">Thank you!</div>
  <div class="w-form-fail" style="display:none">Something went wrong.</div>
</div>
userDetails.js

User Details

Purpose: Fetches the assigned user's full profile and populates page elements dynamically. The central script for personalizing a resource page — manages the profile photo, contact details, booking link, social links, and fires the customBookingLinkUpdated event consumed by other scripts.

How It Works

  1. Reads userId from the URL.
  2. Calls GET https://apiv2.rapidfunnel.com/v2/users-details/{userId}.
  3. Iterates each key in the returned userData object and applies it to the DOM:
Data FieldBehavior
profileImage*Sets src on all [id^=profileImage*] elements. Falls back to default avatar if empty.
firstName / lastNamePopulates elements by ID and by .firstName / .lastName classes.
emailSets href="mailto:..." and text on #email.
phoneNumberSets href="tel:..." on #phoneNumber. Hides parent element if empty.
customBookingLinkSets href on #customBookingLink and .custom_custombookinglink. Fires customBookingLinkUpdated event. Hides element if empty.
Social link fields (e.g., facebookUrl)Sets href on matching .footer-social-links a#[fieldName]. Hides the link if the value is empty.
All other fieldsSets text content on #{fieldName} if the element exists.
  1. Stores customBookingLink in window.sharedData.customBookingLink for cross-script access.
  2. Suppresses Font Awesome pseudo-content for email and phone icons to prevent duplicate rendering.

Custom Event Fired

customBookingLinkUpdated — dispatched on document after the booking link is set. Consumed by bookMe.js and videoScript.js.

Dependencies

jQuery URL param: userId

Required HTML

Elements with IDs matching RapidFunnel user data field names. Common examples:

<img  id="profileImage">
<span id="firstName"></span>
<span id="lastName"></span>
<a    id="email"></a>
<a    id="phoneNumber"></a>
<a    id="customBookingLink" href="#"></a>

<!-- Social links -->
<div class="footer-social-links">
  <a id="facebookUrl"  href="#"></a>
  <a id="twitterUrl"   href="#"></a>
  <a id="linkedinUrl"  href="#"></a>
</div>
videoScript.js

Video Script (Wistia)

Purpose: Integrates Wistia-embedded videos with RapidFunnel analytics tracking. Pushes play events and watch-percentage data to the SQS queue, and controls "Book Me" container visibility based on how long the visitor has watched.

How It Works

  1. Adds Wistia embed classes to any <iframe> whose src contains "wistia".
  2. Listens for customBookingLinkUpdated event to toggle #requestCallContainer visibility.
  3. If showBookMeAfterTimeInSecondsPassedInVideo > 0, hides #bookMeContainer on load.
  4. Uses the Wistia _wq API to bind to all embedded videos:
    • play event: Captures metadata (hashedId, duration, visitorKey, eventKey) and POSTs to /landing/resource/push-to-sqs.
    • secondchange event: Shows #bookMeContainer once secondsWatched ≥ showBookMeAfterTimeInSecondsPassedInVideo.
  5. Only activates tracking if contactId, userId, and resourceId are all valid numerics.

Dependencies

jQuery Wistia embed script userDetails.js URL params: userId, resourceId, contactId Global var: showBookMeAfterTimeInSecondsPassedInVideo

Required HTML

Wistia <iframe> embeds, plus:

<div id="bookMeContainer">...</div>
<div id="requestCallContainer">...</div>
<a   id="customBookingLink" href="#"></a>
<input id="contactEmail" type="hidden">     <!-- optional, for Wistia email tracking -->
<input id="webinar"       type="hidden">     <!-- optional -->
